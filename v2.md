## v2 Changes Summary

### Database Schema Updates
- Added `number` field to players table
- Added `season_type` (regular/playoff) to games table
- Changed role constraint from 'goalkeeper' to 'goalie'
- Updated unique constraint on games to include season_type

### UI Improvements
- New tabbed interface with dedicated "Edit Game" tab
- Player roster shows in a treeview with ID, Name, Role, Number columns
- Edit player section allows updating existing player details
- Shots field is now auto-calculated (Saves + GA) and read-only
- Season type selector (regular/playoff) in game entry
- Dashboard is read-only and shows filtered stats by season type
- Better error messages for duplicate game entries

### Backend Changes
- Auto-send email notification after saving a game
- Full game editing capability (load, modify, save)
- Support for listing games by season and filtering by season_type
- All "goalkeeper" terminology changed to "goalie"

### Migration Note
The v1 database has been backed up to `data/hockey_v1_backup.sqlite3`. 
A new database will be created with the updated schema on first run.
